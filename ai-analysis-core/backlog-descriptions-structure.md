# Правила структуры папок описаний бэклога

## Назначение
Данная структура папок предназначена для хранения детальных описаний каждого элемента бэклога в формате Markdown. Это позволяет разделять общую структуру (хранимую в `backlog.json`) и подробную документацию/спецификации по каждой задаче.

## Правила генерации
1. **Корневая папка**: `documents/backlog-descriptions/`
2. **Иерархия**: Полностью повторяет иерархию из `documents/backlog.json`.
3. **Именование папок**: `[ID] [Title]`
   - Пример: `E4 Администрирование и Безопасность (Admin & Security)`
   - Пример: `F4.1 Управление пользователями (RBAC)`
4. **Содержимое**: В каждой папке должен находиться файл `description.md`.

## Автоматизация
При изменении `backlog.json` (добавлении, удалении или переименовании элементов) нейросеть должна:
1. Просканировать `backlog.json`.
2. Создать недостающие папки.
3. Переименовать папки, если изменился `title` или `id`.
4. Создать пустой `description.md`, если он отсутствует.
5. (Опционально) Обновить заголовок в `description.md` в соответствии с новым `title`.

## Структура description.md
Каждый файл `description.md` должен начинаться с заголовка первого уровня, содержащего ID и название элемента.
Пример:
```markdown
# E4 Администрирование и Безопасность (Admin & Security)

[Описание из backlog.json]
```
