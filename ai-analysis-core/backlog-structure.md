# Описание структуры бэклога

## Назначение

Файл `backlog.json` предназначен для хранения полной иерархической структуры бэклога проекта. На основании этого файла строится визуализация прогресса и планирование спринтов.

## Иерархическая модель

Бэклог организован в виде древовидной структуры с тремя основными уровнями:

1. **Epic (Эпик)** — Верхний уровень. Группирует крупные блоки функциональности. Имеет глобальный приоритет (поле `priority`).
2. **Feature (Функция)** — Средний уровень. Описывает конкретную бизнес-возможность. Связан с одной или несколькими User Stories. Родитель — Epic.
3. **Task (Задача)** — Нижний уровень. Техническая единица реализации. Родитель — Feature.

## Спецификация JSON-объекта

Каждая запись в бэклоге должна соответствовать следующей структуре:

```json
{
  "id": "string",          // Уникальный идентификатор (например: E1, F1.1, T1.1.1)
  "children": [],          // Вложенные элементы бэклога
  "type": "string",        // Тип элемента: "Epic", "Feature", "Task"
  "priority": "number",    // Приоритет (в основном для Epic)
  "title": "string",       // Краткое наименование
  "description": "string", // Подробное описание цели или требований
  "metadata": {            // Дополнительные атрибуты
    "tags": [],            // Метки для фильтрации (например, ["Backend", "UI", "Git"])
    "requirements_check": [ // Трассировка требований: обоснование появления данного пункта в бэклоге
      {
        "document": "string", // Путь к файлу или название документа-источника (например, "base-requirements.md")
        "texts": [
          "string" // Цитаты или ключевые фразы из документа, которые обосновывают данную задачу/фичу
        ]
      }
    ]
  }
}
```
