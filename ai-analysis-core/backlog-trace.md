# Трассировка требований (Requirements Traceability)

Целью является подтверждение того, что каждое исходное требование нашло свое отражение в бэклоге (прямая трассировка), а каждый элемент бэклога обоснован исходными требованиями (обратная трассировка).

## Предусловия

1. Наличие актуального файла [`documents/backlog.json`](documents/backlog.json).
2. Наличие исходных документов:
   - [`base-requirements.md`](base-requirements.md)
   - [`logic_scheme.md`](logic_scheme.md)
3. Соответствие бэклога структуре, описанной в [`ai-analysis-core/backlog-structure.md`](ai-analysis-core/backlog-structure.md).

## Процесс трассировки

### 1. Подготовка

Агент считывает `backlog.json` и все файлы из раздела "Предусловия".

### 2. Создание артефактов проверки

Для каждого исходного файла создается копия файла `имя_файла_требования.trace.md` (например, `base-requirements.trace.md`).

### 3. Алгоритм проверки (Обратная трассировка)

Для каждого пункта в `backlog.json`

1. Пройти по файлам с требованиями и найти текст, на основании которого был создан этот пункт бэклога.
2. В файле копии `имя_файла_требования.trace.md` зафиксировать связь (после соответствуюего предложения в тексте) :
[
   - **Требование:** [Текст или ID требования]
   - **Статус:** [Покрыто / Частично / Не покрыто]
   - **Элементы бэклога:** [ID элементов, например, E1, F1.1, US1.1.2]
   - **Комментарий:** (если требуется пояснение)
]

### 4. Алгоритм проверки (Прямая трассировка)

Проверить поле `metadata.requirements_check` в `backlog.json`:

1. Убедиться, что для каждого элемента указан существующий документ и корректная цитата.
2. Если элемент бэклога не имеет ссылки на исходное требование, пометить (указать в названии) его как "Unjustified" (необоснованный).

## Критерии успешности

- **100% покрытие:** Все пункты из `base-requirements.md` и `logic_scheme.md` имеют связи с элементами бэклога.
- **Обоснованность:** Нет элементов бэклога (кроме технических задач), которые не ссылаются на исходные требования.
- **Актуальность:** Все ссылки в `metadata.requirements_check` соответствуют реальному тексту в документах.

## Обработка несоответствий

- Если требование **не покрыто**: создать задачу в бэклоге или зафиксировать пропуск в отчете для обсуждения.
- Если требование **избыточно** (не актуально): пометить в исходном файле и отчете трассировки.
