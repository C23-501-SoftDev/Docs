# Участники процессов (Actors)

| Актер | Описание | Основные цели |
| :--- | :--- | :--- |
| **Гость (GUEST)** | Активный пользователь с минимальными правами. Может только войти в систему. Все возможности получает через явное назначение прав на конкретные пространства. При soft-удалении (`is_deleted = true`) теряет возможность входа. | Вход в систему, работа с назначенными пространствами и документами. |
| **Читатель (READER)** | Пользователь с глобальным правом на чтение любых документов. | Поиск информации, чтение документов, экспорт в PDF/DOCX/HTML. |
| **Редактор (EDITOR)** | Пользователь с глобальным правом на создание, редактирование и удаление любых документов. | Создание документов по шаблонам, редактирование, управление версиями, удаление документов. |
| **Администратор (ADMIN)** | Пользователь с флагом `is_admin = true`. Управляет системой, пользователями и правами доступа. Может иметь роль GUEST, READER или EDITOR для работы с контентом. Умеет мягко удалять и восстанавливать пользователей. | Управление пользователями (создание, редактирование, soft-удаление, восстановление), создание пространств (Spaces), назначение прав доступа пользователям и группам, восстановление удаленных документов, настройка конфигураций сред. |
| **Система (System Actor)** | Бэкенд базы знаний, обеспечивающий логику работы приложения. | Обработка запросов, хранение данных, рассылка уведомлений, управление правами доступа, принудительная блокировка soft-удалённых пользователей при попытке входа. |
