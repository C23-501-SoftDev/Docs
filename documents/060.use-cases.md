# Use Case диаграммы и описания (Use Cases)

    Editor --> (Soft-удаление документа)

    Admin --> (Управление пользователями)
    Admin --> (Управление группами пользователей)
    Admin --> (Управление пространствами)

**Примечание:** Администратор (`ADMIN`) — это пользователь с `is_admin = true`. Его права на контент определяются ролью (`GUEST`/`READER`/`EDITOR`). На диаграмме показаны только административные функции, доступные при `is_admin = true`.

## Описание ключевых процессов

### UC-1: Создание документа по шаблону
* **Действующее лицо:** Редактор (пользователь с ролью `EDITOR`) или пользователь с правом `WRITE`/`OWNER` в пространстве.
* **Предусловие:** Пользователь авторизован в системе.
* **Основной поток:**
    1. Редактор нажимает кнопку "Создать документ".
    2. Система предлагает выбрать тип шаблона (Архитектура, Требования, Инструкция и т.д.).
    3. Редактор выбирает шаблон.
    4. Система открывает форму с предзаполненными атрибутами и полем для Wiki-разметки.
    5. Редактор заполняет данные и выбирает Пространство.
    6. Редактор нажимает "Сохранить".
    7. Система создает документ в статусе `Draft`.
* **Постусловие:** Документ создан и доступен автору.

### UC-2: Управление версиями (Откат)
* **Действующее лицо:** Редактор (пользователь с ролью `EDITOR`) или пользователь с правом `WRITE`/`OWNER` в пространстве.
* **Предусловие:** Документ существует и имеет несколько версий.
* **Основной поток:**
    1. Редактор открывает документ.
    2. Переходит в раздел "История изменений".
    3. Выбирает две версии для сравнения.
    4. Система отображает визуальную разницу (diff).
    5. Редактор нажимает "Восстановить эту версию".
    6. Система создает новую версию документа, идентичную выбранной.
* **Постусловие:** Текущее состояние документа обновлено до выбранной версии.

### UC-3: Управление пространствами
* **Действующее лицо:** Администратор (пользователь с `is_admin = true`).
* **Предусловие:** Администратор авторизован.
* **Основной поток:**
    1. Администратор переходит в раздел "Управление пространствами".
    2. Нажимает "Создать пространство".
    3. Вводит название, описание и назначает ответственного редактора.
    4. Настраивает права доступа для пользователей и групп (Чтение/Запись).
    5. Сохраняет изменения.
* **Постусловие:** Новое пространство создано и готово к наполнению документами.

### UC-4: Назначение прав доступа
* **Действующее лицо:** Администратор (пользователь с `is_admin = true`).
* **Предусловие:** Администратор авторизован.
* **Основной поток:**
    1. Администратор открывает настройки пространства.
    2. Выбирает пользователя или группу из списка.
    3. Назначает уровень доступа (`READ`, `WRITE`, `OWNER`).
    4. Сохраняет изменения.
* **Постусловие:** Права доступа на пространство обновлены. Затронутые пользователи получают уведомления.

### UC-5: Управление группами пользователей
* **Действующее лицо:** Администратор (пользователь с `is_admin = true`).
* **Предусловие:** Администратор авторизован в системе.
* **Основной поток:**
    1. Администратор переходит в раздел "Группы пользователей" в панели администратора.
    2. Система отображает таблицу существующих групп (название, описание, количество участников).
    3. Администратор может:
        * **Создать группу:** Нажать "Создать группу" → ввести название и описание → сохранить. Система создает группу, группа отображается в таблице.
        * **Редактировать группу:** Нажать на группу → изменить название или описание → сохранить.
        * **Удалить группу:** Нажать "Удалить" → подтвердить действие. Система удаляет группу, все связи с пользователями и права группы на пространства удаляются каскадно. Пользователи не затрагиваются.
        * **Управлять составом:** Нажать на группу → перейти на страницу состава. Администратор видит список участников группы и выпадающий список всех пользователей системы. Может добавить пользователя в группу или исключить из неё.
* **Альтернативный поток (назначение прав группе):**
    1. Администратор переходит в раздел "Управление пространствами".
    2. Выбирает пространство → "Настройка прав".
    3. В интерфейсе назначения прав выбирает вкладку "Группы" (помимо вкладки "Пользователи").
    4. Выбирает группу из списка → назначает уровень доступа (`READ`, `WRITE`, `OWNER`) → сохраняет.
* **Постусловие:** Группы созданы/изменены. Права, полученные пользователями через группы, начинают действовать немедленно (при следующем запросе система вычисляет максимальное право).
