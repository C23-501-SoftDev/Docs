# Матрица ролей и прав доступа (Role Matrix / RBAC)

Система использует ролевую модель доступа (RBAC) с дополнительными ограничениями на уровне Пространств и Документов.

| Действие | Читатель | Редактор | Администратор | Примечание |
| :--- | :---: | :---: | :---: | :--- |
| **Авторизация** | + | + | + | |
| **Поиск и чтение документов** | + | + | + | Только в разрешенных Пространствах |
| **Навигация по оглавлению** | + | + | + | |
| **Экспорт документов** | + | + | + | |
| **Создание документа** | - | + | + | Редактор создает в своих Пространствах |
| **Редактирование документа** | - | + | + | Редактор — если автор или есть права |
| **Управление вложениями** | - | + | + | |
| **Просмотр истории версий** | + | + | + | |
| **Откат версий** | - | + | + | |
| **Soft-удаление документа** | - | + | + | |
| **Восстановление из корзины** | - | + | + | |
| **Hard-delete документа** | - | - | + | |
| **Управление пользователями** | - | - | + | |
| **Управление пространствами** | - | - | + | |
| **Настройка конфигураций** | - | - | + | |

## Уровни доступа

1. **Глобальный уровень (Роль):** Определяет базовый набор доступных функций (кнопок в UI).
2. **Уровень Пространства (Space):** Администратор назначает права группам пользователей на чтение или запись внутри конкретного пространства.
3. **Уровень Документа:** Автор документа может дополнительно ограничить или расширить доступ к конкретному документу (в рамках прав Пространства).
