# Матрица ролей и прав доступа (Role Matrix / RBAC)

Система использует комбинированную модель доступа: глобальная роль (`GUEST`, `READER`, `EDITOR`) + флаг администратора (`is_admin`) + дополнительные права на уровне Пространств (личные и групповые) и Документов.

## Глобальные роли

| Роль | Описание |
| :--- | :--- |
| **GUEST** | Базовая роль. Может только войти в систему. Все права на контент получает через назначение личных прав, прав через группы на пространства и прав на документы. |
| **READER** | Глобальное право на чтение любых документов в системе. |
| **EDITOR** | Глобальное право на создание, редактирование и удаление любых документов в системе. |
| **ADMIN** | Флаг `is_admin = true`. Доступ к панели администратора, управление пользователями, группами, пространствами и правами доступа. Может комбинироваться с любой из трех ролей. |

**Примечания:**
1. `ADMIN` — это флаг, а не отдельная роль. Пользователь с `is_admin = true` всегда имеет одну из трех ролей (`GUEST`/`READER`/`EDITOR`), которая определяет его права на контент. Для входа в систему требуется наличие роли.
2. `GUEST` получает права на чтение/запись через механизм назначения личных прав на пространства (`SpaceUserPermission`), прав через группы (`SpaceGroupPermission`) и прав на документы (`DocumentPermission`).
3. `ADMIN` без роли `READER` или `EDITOR` (т.е. с ролью `GUEST`) не имеет глобальных прав на контент — только административные функции (включая управление группами). Права на контент могут быть получены через назначение прав на пространства/документы.
4. `Hard-delete` доступен только пользователям с `is_admin = true`, независимо от их роли.

## Уровни доступа

1. **Глобальный уровень (Роль + is_admin):** Определяет базовый набор доступных функций и видимость элементов UI.
2. **Уровень Пространства (Space):** Администратор или владелец назначает права пользователям (лично) и группам на чтение (`READ`), запись (`WRITE`) или владение (`OWNER`) внутри конкретного пространства. Для каждого пользователя итоговое право вычисляется как максимальное из глобальной роли, личных прав и прав всех групп, в которых он состоит.
3. **Уровень Документа:** Автор документа или администратор может дополнительно ограничить или расширить доступ к конкретному документу (в рамках прав Пространства).

## Комбинации ролей

| role | is_admin | Типичный представитель |
| :--- | :---: | :--- |
| `GUEST` | `false` | Новый сотрудник, внешний консультант — получает доступ только к назначенным пространствам |
| `GUEST` | `true` | Системный администратор — управляет пользователями, группами и правами, но не работает с контентом |
| `READER` | `false` | Сотрудник, которому нужно читать документацию |
| `READER` | `true` | Старший администратор — читает документацию, управляет системой и группами |
| `EDITOR` | `false` | Технический писатель, аналитик — создает и редактирует документы |
| `EDITOR` | `true` | Главный редактор с полным доступом — управляет контентом, группами и системой |
