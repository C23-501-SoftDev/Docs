# Описание US4.1.2: Создание сущности User и JWT авторизация

## Описание

US4.1.2 является реализацией подсистемы идентификации в рамках **Feature 4.1: Управление пользователями (RBAC)** (Epic 4). Она обеспечивает выполнение требований по аутентификации пользователя по паре "логин/пароль" и является необходимым условием для работы разграничения прав доступа во всех остальных модулях системы.

## Технические задачи реализации

1. **Модель Пользователя:**
   - Реализация сущности `User` в PostgreSQL 18.
   - Обеспечение валидации входящих данных (Constraint на email, уникальность логина).
2. **Сервис аутентификации:**
   - Интеграция подсистемы **Issuer** (генерация JWT) и **Validator** (проверка JWT при каждом запросе).
   - Хранение паролей с использованием алгоритма BCrypt.
3. **Безопасность (API & RBAC):**
   - Настройка Spring Security для защиты RESTful API согласно спецификации OpenAPI.

## Критерии приемки (Acceptance Criteria)

* **Сценарий 1: Аутентификация**
    * **Given** пользователь ввел корректный логин и пароль
    * **When** отправлен запрос `/api/auth/login`
    * **Then** система возвращает JWT-токен, который учитывается в последующих запросах (Authorization: Bearer <token>)
* **Сценарий 2: Инициализация уровня доступа**
    * **Given** пользователь обладает ролью `Reader`
    * **When** он пытается выполнить CRUD-операцию изменения документа (Feature 1.1)
    * **Then** система возвращает ошибку 403 Forbidden, основываясь на валидации JWT
* **Сценарий 3: Безопасность данных**
    * **Given** создание новой учетной записи
    * **Then** пароль сохраняется в БД строго в шифрованном виде
* **Сценарий 4: Валидация сессии**
    * **Given** токен истек
    * **When** выполняется запрос к защищенному ресурсу
    * **Then** система отклоняет запрос, возвращая ошибку 401 Unauthorized
