# US4.1.5 Логирование действий системы

## Описание
US4.1.5 завершает функционал **Feature 4.1: Управление пользователями (RBAC)** (Epic 4), реализуя систему аудита. Она необходима для отслеживания всех критических изменений в базе знаний и обеспечения прозрачности работы системы для требований безопасности и диагностики ошибок.

## Технические задачи реализации
1. **Система аудита:**
   - Реализация перехватчиков (interceptors/AOP) для логирования ключевых действий пользователей (создание, изменение, удаление документов, изменение прав доступа).
2. **Структура логов:**
   - Сохранение в БД записей с указанием: `timestamp`, `user_id`, `action_type`, `resource_id`, `ip_address`.
3. **Безопасность логов:**
   - Обеспечение доступа к журналу событий только для пользователей с ролью `Admin`.

## Критерии приемки (Acceptance Criteria)
* **Сценарий 1: Аудит операций над документами**
    * **Given** пользователь выполнил действие (создание/изменение/удаление документа)
    * **When** событие завершено
    * **Then** система создает запись в журнале аудита с указанием автора и времени действия
* **Сценарий 2: Журналирование административных операций**
    * **Given** Администратор изменил права доступа для пользователя или пространства
    * **When** действие подтверждено
    * **Then** в журнале появляется соответствующая запись с указанием старых и новых прав
* **Сценарий 3: Доступность журнала**
    * **Given** пользователь с ролью `Reader` или `Editor` пытается просмотреть системные логи
    * **Then** система отклоняет доступ с ошибкой 403 Forbidden
* **Сценарий 4: Поиск по логам**
    * **Given** Администратор открыл панель логирования
    * **When** он вводит фильтр по конкретному `user_id` или дате
    * **Then** система отображает отфильтрованный список событий, совершенных указанным пользователем
