# US4.1.8 Создание и управление группами

## Описание
Реализация функционала группировки пользователей для упрощения администрирования доступа.

## Технические требования
* **Backend:** API для создания, чтения, обновления и удаления (CRUD) сущности `Group`.
* **Database:** Таблица `groups` в PostgreSQL, связующая таблица для связи групп и пользователей.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Создание группы
* **Given:** Администратор находится в панели управления группами.
* **When:** Администратор создает группу с уникальным названием.
* **Then:** Группа появляется в системе и доступна для дальнейшего управления.

### Сценарий 2: Удаление группы
* **Given:** В системе существует группа.
* **When:** Администратор удаляет группу.
* **Then:** Группа удаляется, права доступа, зависящие от этой группы, должны быть пересмотрены (или отозваны).
