# US4.1.1 Инициализация проекта

## Описание

Первичная настройка технологического стека и инфраструктуры проекта «База знаний» для обеспечения дальнейшей разработки функциональных модулей.

## Технические требования

### 1. Бэкенд (Spring Boot & Spring Web MVC)

* **Каркас приложения:** Инициализация проекта на Java с использованием Spring Boot.
* **Веб-фреймворк:** Настройка Spring Web MVC в качестве основного каркаса веб-приложения (настройка DispatcherServlet, MVC-контроллеров, представлений/шаблонов и статических ресурсов). Реализация базовой стартовой страницы (Index Page), доступной по корневому URL (`/`).
* **Система сборки:** Настройка Maven (pom.xml) с подключением необходимых зависимостей (Spring Web MVC, Spring Data JPA, Spring Security, Thymeleaf/FreeMarker для UI, PostgreSQL Driver и др.).
* **Архитектура:** Реализация базовой структуры пакетов в соответствии с принципами Чистой архитектуры (Clean Architecture).
* **API:** Настройка OpenAPI/Swagger для документирования RESTful API.

### 2. База данных (PostgreSQL)

* **Версия:** Развертывание и настройка PostgreSQL 18.
* **Миграции:** Настройка инструмента миграции БД (Liquibase или Flyway) для управления схемой данных.
* **Подключение:** Настройка пула соединений в `application.properties`/`application.yml`.

### 3. Окружение и логирование

* **Логирование:** Настройка логирования действий системы (Logback/SLF4J).
* **Профили:** Настройка профилей окружения (dev, test, prod).

### 4. Безопасность и Ролевая модель (Spring Security)

* **Конфигурация Security:** Настройка Spring Security для Spring MVC (SecurityFilterChain) для защиты веб-страниц и API-эндпоинтов приложения.
* **Схема БД для RBAC:** Создание миграций для инициализации базовых таблиц ролевой модели: `users`, `roles`, `user_roles`.
* **Инициализация данных (Seed Data):** Автоматическое создание начальных ролей (`ADMIN`, `USER`) и дефолтного пользователя-администратора при первичном запуске.
* **Аутентификация:** Настройка базового механизма входа (Form Login) для доступа к защищенным разделам приложения.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Сборка и запуск приложения

* **Given:** Исходный код проекта склонирован, установлены JDK и Maven.
* **When:** Выполняется команда `mvn clean install` и последующий запуск приложения командой `mvn spring-boot:run`.
* **Then:** Проект собирается без ошибок, приложение запускается, по корневому адресу `/` открывается стартовая страница, а по адресу `/swagger-ui.html` доступна документация API.

### Сценарий 2: Подключение к базе данных

* **Given:** Сервер PostgreSQL 18 запущен и создана целевая БД.
* **When:** Приложение инициализируется.
* **Then:** Соединение с БД успешно установлено, и применены начальные скрипты миграции.

### Сценарий 3: Инициализация БД ролевой модели и пользователя-администратора

* **Given:** Пустая база данных приложения.
* **When:** Приложение успешно запускается.
* **Then:** В БД созданы таблицы `users`, `roles`, `user_roles`. В таблицах присутствует роль `ADMIN` и активный пользователь-администратор.

### Сценарий 4: Проверка ролевого доступа (RBAC) в Spring MVC

* **Given:** Приложение запущено, настроены защищенные эндпоинты.
* **When:** Анонимный пользователь пытается получить доступ к защищенному разделу.
* **Then:** Система перенаправляет пользователя на страницу авторизации.
* **When:** Пользователь авторизуется под учетной записью администратора.
* **Then:** Доступ к защищенному разделу предоставляется успешно.
