# US4.1.1 Инициализация проекта

## Описание
Первичная настройка технологического стека и инфраструктуры проекта «База знаний» для обеспечения дальнейшей разработки функциональных модулей.

## Технические требования

### 1. Бэкенд (Spring Boot)
* **Каркас приложения:** Инициализация проекта на Java с использованием Spring Boot.
* **Система сборки:** Настройка Maven (pom.xml) с подключением необходимых зависимостей (Spring Web, Spring Data JPA, Security и др.).
* **Архитектура:** Реализация базовой структуры пакетов в соответствии с принципами Чистой архитектуры (Clean Architecture).
* **API:** Настройка OpenAPI/Swagger для документирования RESTful API.

### 2. База данных (PostgreSQL)
* **Версия:** Развертывание и настройка PostgreSQL 18.
* **Миграции:** Настройка инструмента миграции БД (например, Liquibase или Flyway) для управления схемой данных.
* **Подключение:** Настройка пула соединений в `application.properties`/`application.yml`.

### 3. Файловое хранилище и Git
* **SMB Интеграция:** Настройка доступа к файловой системе по протоколу SMB для хранения Wiki-файлов и вложений.
* **Git Server-side:** Инициализация Git-репозитория на сервере для обеспечения версионности документов.
* **Связка:** Реализация базового механизма взаимодействия сервера с SMB-шарой через Java API.

### 4. Окружение и логирование
* **Логирование:** Настройка логирования действий системы (Logback/SLF4J).
* **Профили:** Настройка профилей окружения (dev, test, prod).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Сборка и запуск приложения
* **Given:** Исходный код проекта склонирован, установлены JDK и Maven.
* **When:** Выполняется команда `mvn clean install` и последующий запуск приложения.
* **Then:** Проект собирается без ошибок, приложение запускается, и по адресу `/swagger-ui.html` (или аналогичному) доступна документация API.

### Сценарий 2: Подключение к базе данных
* **Given:** Сервер PostgreSQL 18 запущен и создана целевая БД.
* **When:** Приложение инициализируется.
* **Then:** Соединение с БД успешно установлено, и применены начальные скрипты миграции (если они есть).

### Сценарий 3: Доступ к файловому хранилищу
* **Given:** Настроена SMB-директория с правами доступа для сервисного аккаунта.
* **When:** Приложение обращается к файловой системе.
* **Then:** Приложение успешно выполняет операции чтения и записи тестового файла в целевой директории.

### Сценарий 4: Инициализация Git-репозитория
* **Given:** Целевая директория для хранения документов доступна.
* **When:** Выполняется процедура инициализации инфраструктуры.
* **Then:** В целевой директории создан валидный Git-репозиторий (наличие папки `.git`), готовый к фиксации изменений.
