# US4.1.1 Инициализация проекта

## Описание
Первичная настройка технологического стека и инфраструктуры проекта «База знаний» для обеспечения дальнейшей разработки функциональных модулей.

## Технические требования
* **Стек/Инструменты:** Java 17+, Spring Boot, Spring Data JPA, Maven, PostgreSQL 18, JGit, OpenAPI/Swagger, Logback/SLF4J.
* **Архитектурные аспекты:** Реализация базовой структуры пакетов в соответствии с принципами Чистой архитектуры (Clean Architecture). Подготовка каркаса для многослойного взаимодействия (контроллеры → сервисы → репозитории).
* **Инфраструктура/Конфигурация:**
  * Развертывание PostgreSQL 18, создание базы данных `knowledge_base`.
  * Использование инструмента миграции БД (например, Liquibase или Flyway) для управления схемой данных.
  * Инициализация локальной директории для Git-репозитория (хранение `.md`-файлов документов).
  * Инициализация отдельной локальной директории для blob-хранилища (изображения, вложения).
  * Настройка профилей окружения (`dev`, `test`, `prod`) в `application.properties`.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Сборка и запуск приложения
* **Given:** Исходный код проекта склонирован, установлены JDK 17+ и Maven.
* **When:** Выполняется команда `mvn clean install` и последующий запуск приложения.
* **Then:** Проект собирается без ошибок, приложение запускается, и по адресу `/swagger-ui.html` доступна документация API.

### Сценарий 2: Подключение к базе данных
* **Given:** Сервер PostgreSQL 18 запущен и создана целевая БД `knowledge_base`.
* **When:** Приложение инициализируется.
* **Then:** Соединение с БД успешно установлено, и применены начальные скрипты миграции (если они есть).

### Сценарий 3: Доступ к Git-репозиторию
* **Given:** Инициализирована директория для Git-репозитория.
* **When:** Приложение через JGit создаёт тестовый коммит (пустой или с тестовым `.md`-файлом).
* **Then:** Коммит успешно создаётся, и в репозитории появляется новый объект коммита.

### Сценарий 4: Доступ к blob-хранилищу
* **Given:** Инициализирована директория для blob-хранилища.
* **When:** Приложение записывает тестовый бинарный файл и читает его обратно.
* **Then:** Файл успешно записывается и читается без ошибок.
