# US4.1.4 Панель администратора (CRUD пользователей)

## Описание

Разработка административного интерфейса для управления учетными записями пользователей (создание, редактирование, удаление) и назначения им глобальных ролей (`Admin`, `Editor`, `Reader`).

### Ограничения на удаление пользователя

В системе действуют ограничения целостности данных (`ON DELETE RESTRICT`):
- **Нельзя удалить пользователя**, если он является автором хотя бы одного документа (`documents.author_id`)
- **Нельзя удалить пользователя**, если он является владельцем хотя бы одного пространства (`spaces.owner_id`)
- **Нельзя удалить пользователя**, если он создал хотя бы одну версию документа (`versions.author_id`)

При попытке удаления такого пользователя система должна отклонить операцию с соответствующим сообщением.

## Технические требования

### Стек/Инструменты
- **Бэкенд:** Java/Spring Boot (REST API)
- **Фронтенд:** React (UI панели администратора)

### API эндпоинты (префикс `/api/admin/users`)

| Метод | Эндпоинт | Описание | Доступ |
|-------|----------|----------|--------|
| GET | `/api/admin/users` | Получение списка всех пользователей (с пагинацией) | Admin |
| GET | `/api/admin/users/{id}` | Получение информации о конкретном пользователе | Admin |
| POST | `/api/admin/users` | Создание нового пользователя (логин, email, пароль, роль) | Admin |
| PUT | `/api/admin/users/{id}` | Обновление данных пользователя (роль, email, логин) | Admin |
| DELETE | `/api/admin/users/{id}` | Удаление пользователя (с учётом ограничений `ON DELETE RESTRICT`) | Admin |
| PUT | `/api/admin/users/{id}/password` | Сброс/изменение пароля пользователя | Admin |

### Архитектурные аспекты
- Все эндпоинты защищены: доступ только для пользователей с глобальной ролью `Admin`
- При создании пользователя пароль хешируется с использованием BCrypt
- При обновлении роли пользователя изменения сохраняются в БД, но вступают в силу **только после повторной аутентификации** пользователя (создания нового JWT-токена), так как роль закодирована в текущем токене

### Инфраструктура/Конфигурация
- Доступ к панели администратора в UI ограничен ролью `Admin` (фронтенд скрывает пункт меню/роуты для не-администраторов)

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Управление пользователем (создание и редактирование)
- **Given:** Учетная запись администратора
- **When:** Администратор создаёт нового пользователя или обновляет карточку существующего (изменяет роль, email или логин)
- **Then:** Изменения успешно сохраняются в БД. При изменении роли пользователь получит новые права **после повторного входа в систему** (новый JWT-токен)

### Сценарий 2: Смена пароля пользователя администратором
- **Given:** Администратор находится на странице редактирования пользователя
- **When:** Администратор указывает новый пароль и сохраняет изменения
- **Then:** Система хеширует новый пароль (BCrypt) и сохраняет в поле `password_hash`. Пользователь может войти с новым паролем при следующей аутентификации

### Сценарий 3: Удаление пользователя (без связанных данных)
- **Given:** Учетная запись пользователя существует и **не имеет связанных документов, пространств или версий** (пользователь не является автором документов, владельцем пространств и не создавал версий)
- **When:** Администратор удаляет учетную запись
- **Then:** Учетная запись физически удаляется из БД

### Сценарий 4: Отказ в удалении пользователя (есть связанные данные)
- **Given:** Учетная запись пользователя существует **и имеет связанные документы** (является автором) или **является владельцем пространства**, или **создавал версии документов**
- **When:** Администратор пытается удалить учетную запись
- **Then:** Система отклоняет операцию с сообщением о невозможности удаления (например: «Невозможно удалить пользователя: он является автором документов»)

### Сценарий 5: Отказ в доступе к панели администратора (не-Admin)
- **Given:** Пользователь с глобальной ролью `Editor` или `Reader` (не `Admin`)
- **When:** Пользователь пытается открыть URL панели администратора в UI или вызвать любой из API-эндпоинтов `/api/admin/*`
- **Then:** Система возвращает ошибку 403 Forbidden, а UI скрывает все административные элементы интерфейса
