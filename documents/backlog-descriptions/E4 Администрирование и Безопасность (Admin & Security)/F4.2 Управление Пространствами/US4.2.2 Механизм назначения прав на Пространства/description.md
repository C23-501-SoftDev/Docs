# US4.2.2 Механизм назначения прав на Пространства

## Описание
Реализация функциональности управления доступом пользователей к Пространствам, включая назначение ролей для определенных групп пользователей в выбранном Space.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Security), БД (связи User-Space-Role).
* **Архитектурные аспекты:** Модель RBAC масштабирована на уровень отдельных пространств (иерархия доступа).
* **Инфраструктура/Конфигурация:** Таблица-связка доступа с правами.
* **Уровни доступа в Пространстве:**
    * **READ (Чтение):** Просмотр списка документов и их содержимого, экспорт.
    * **WRITE (Запись):** Создание новых документов, редактирование существующих, управление вложениями.
    * **OWNER (Владелец):** Полный контроль над пространством, включая удаление документов и управление правами других пользователей внутри этого Space.

## UX/UI Требования к поиску
* **Автодополнение:** При назначении прав поиск пользователя или группы должен осуществляться через выпадающий список.
* **Триггер поиска:** Список должен появляться после ввода минимум 3-х символов.
* **Debounce:** Реализовать задержку (debounce) при вводе текста перед отправкой запроса на сервер для оптимизации нагрузки.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Успешное назначение
* **Given:** Администратор открывает настройки пространства.
* **When:** Назначает группе пользователей доступ (например, "Reader" для конкретного Space).
* **Then:** Сохраняются соответствующие метаданные прав в БД, фильтрация контента в пространстве применяется автоматически для данной группы.
