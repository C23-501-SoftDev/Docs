# US4.2.2 Механизм назначения прав на Пространства

## Описание
Реализация функциональности управления доступом пользователей к Пространствам, включая назначение ролей для определенных групп пользователей в выбранном Space.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Security), БД (связи User-Space-Role).
* **Архитектурные аспекты:** Модель RBAC масштабирована на уровень отдельных пространств (иерархия доступа).
* **Инфраструктура/Конфигурация:** Таблица-связка доступа с правами (Read/Write/Delete).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Успешное назначение
* **Given:** Администратор открывает настройки пространства.
* **When:** Назначает группе пользователей доступ (например, "Reader" для конкретного Space).
* **Then:** Сохраняются соответствующие метаданные прав в БД, фильтрация контента в пространстве применяется автоматически для данной группы.
