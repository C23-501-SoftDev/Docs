# US4.2.1 CRUD сущности Space

## Описание
Реализация базовых операций (Create, Read, Update, Delete) для управления Пространствами (Spaces) в системе Базы знаний.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot, PostgreSQL 18.
* **Архитектурные аспекты:** Entity `Space` с набором атрибутов (имя, описание, владелец). REST API для выполнения манипуляций со списком пространств.
* **Инфраструктура/Конфигурация:** Валидация прав доступа (только Admin).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Создание пространства
* **Given:** Пользователь авторизован как Admin.
* **When:** Заполняет форму создания новой сущности `Space` и отправляет запрос.
* **Then:** Новое пространство создано и доступно в системе.

### Сценарий 2: Удаление пространства
* **Given:** Пространство существует.
* **When:** Администратор выполняет операцию удаления.
* **Then:** Сущность `Space` удаляется из системы, ресурсы пространства (документы) могут быть перемещены или архивированы.
