# US4.2.1 CRUD сущности Space

## Описание
Реализация базовых операций (Create, Read, Update, Delete) для управления Пространствами (Spaces) в системе Базы знаний.

## Макет (Prototype)
[Открыть макет управления Пространствами](../../../../prototypes/admin-panel/spaces.html) — прототип с таблицей пространств, формой создания/редактирования, пагинацией, сортировкой и симуляцией удаления. Реализована ссылка на страницу управления правами доступа к пространству (Permissions).

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot, PostgreSQL 18.
* **Атрибутивная модель Entity `Space`:**
    * `id`: UUID или Long (первичный ключ).
    * `name`: Строка (обязательное, уникальное значение).
    * `description`: Текст (необязательное описание).
    * `ownerId`: Ссылка на сущность `User` (владелец пространства).
    * `isDeleted`: Boolean (флаг мягкого удаления, по умолчанию false).
    * `createdAt` / `updatedAt`: Временные метки создания и обновления.
* **Архитектурные аспекты:**
    * REST API для управления списком пространств (Admin endpoints).
    * Автоматическое создание записи в `SpacePermission` с типом `OWNER` для создателя/владельца при создании пространства.
* **Инфраструктура/Конфигурация:** Валидация прав доступа (только Admin имеет полный доступ к CRUD).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Создание пространства
* **Given:** Пользователь авторизован как Admin.
* **When:** Заполняет форму создания новой сущности `Space` (имя, описание, опционально владелец) и отправляет запрос.
* **Then:** 
    * Проверяется уникальность имени (при дубликате — ошибка 409 Conflict).
    * Новое пространство создано в БД.
    * Владельцу автоматически выданы права `OWNER` в таблице разрешений.

### Сценарий 2: Редактирование и передача владения
* **Given:** Пространство существует.
* **When:** Admin обновляет описание или меняет `ownerId`.
* **Then:** Данные обновляются, поле `updatedAt` актуализируется.

### Сценарий 3: Soft-удаление пространства
* **Given:** Пространство существует.
* **When:** Администратор выполняет операцию удаления.
* **Then:** 
    * Поле `isDeleted` устанавливается в `true`.
    * Пространство исключается из списков и поиска.
    * Все связанные ресурсы (документы, права доступа) сохраняются для истории и аудита.

### Сценарий 4: Восстановление пространства
* **Given:** Пространство находится в состоянии soft-delete (`isDeleted = true`).
* **When:** Администратор выполняет операцию восстановления.
* **Then:** 
    * Поле `isDeleted` устанавливается в `false`.
    * Пространство снова отображается в списках и доступно для работы.
