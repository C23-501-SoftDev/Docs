# US4.2.3 Назначение прав доступа для групп

## Описание
Привязка политик доступа к пространствам (Spaces) на уровне групп пользователей, а не только отдельных аккаунтов.

## Технические требования
* **Logic:** Модификация логики `SpacePermission`, чтобы поддерживать запись `groupId` помимо `userId`.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Назначение прав группе
* **Given:** Администратор настраивает права для Пространства.
* **When:** Администратор выбирает группу и назначает ей роль (например, READER).
* **Then:** Все пользователи этой группы получают доступ к пространству в соответствии с назначенной ролью.

### Сценарий 2: Динамическое обновление прав
* **Given:** Группа имеет доступ к пространству.
* **When:** Новый пользователь добавляется в группу.
* **Then:** Пользователь автоматически получает доступ к пространству согласно правам группы.
