# US4.3.2 Настройка параметров SMTP и асинхронной рассылки

## Описание
Предоставление администратору возможности конфигурировать параметры почтового сервера для отправки системных рассылок и уведомлений.

## Технические требования
* **Backend:** Хранение конфигурации SMTP (host, port, username, password, ssl/tls) в БД или конфигурационном файле, доступном через панель администрирования.
* **Infrastructure:** Использование асинхронной очереди для отправки сообщений (не блокировать основной поток выполнения при отправке Email).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Успешная настройка SMTP
* **Given:** Администратор находится в панели конфигурации системы.
* **When:** Администратор вводит параметры SMTP-сервера и нажимает "Сохранить".
* **Then:** Система сохраняет настройки и выполняет проверку подключения к серверу.

### Сценарий 2: Отправка тестового письма
* **Given:** SMTP-конфигурация сохранена.
* **When:** Администратор нажимает кнопку "Отправить тестовое письмо".
* **Then:** Система инициирует асинхронную отправку письма на административный Email и отображает статус операции.
