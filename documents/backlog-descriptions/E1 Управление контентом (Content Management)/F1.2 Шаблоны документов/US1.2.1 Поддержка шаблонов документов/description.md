# US1.2.1 Поддержка шаблонов документов

## Описание
Внедрение системы шаблонов документации. Каждый шаблон помечается специализацией (например, "Аналитик", "Разработчик"), но это является лишь информативной отметкой и не ограничивает доступ. Шаблоны для просмотра доступны всем пользователям, имеющим право доступа к системе (guest, reader, editor). Каждый пользователь, обладающий правом на создание документа в конкретном пространстве (согласно матрице прав и настройкам пространства), может воспользоваться предоставленным шаблоном, после чего созданный документ становится обычным документом без привязки к шаблону.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend), Frontend (UI шаблонов).
* **Архитектурные аспекты:** 
    * Хранение шаблонов в БД, где для каждого шаблона сохраняется его специализация как текстовая метка.
    * Сервис получения шаблонов (метод `getAvailableTemplates()`), возвращающий список доступных шаблонов.
* **Инфраструктура/Конфигурация:** Шаблоны предзагружены в систему с метаданными специализации.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Доступ к шаблонам для авторизованных пользователей
* **Given:** Пользователь авторизован в системе.
* **When:** Заходит в режим создания нового документа в пространстве, где у него есть право `canWrite` (согласно `PermissionService.canWrite`).
* **Then:** Система отображает все доступные в системе шаблоны.

### Сценарий 2: Создание документа на основе шаблона
* **Given:** Пользователь выбирает любой шаблон из списка.
* **When:** Пользователь подтверждает создание документа.
* **Then:** Создается новый экземпляр документа, который не сохраняет связь с шаблоном и доступен для редактирования как обычный документ, подчиняющийся общим правилам прав доступа пространства.
