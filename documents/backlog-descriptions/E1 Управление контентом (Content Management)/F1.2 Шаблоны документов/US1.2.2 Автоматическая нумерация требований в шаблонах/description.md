# US1.2.2 Автоматическая нумерация требований в шаблонах

## Описание
Реализация механизма автоматической генерации уникальных номеров для требований внутри документов, созданных из шаблонов.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend Service).
* **Архитектурные аспекты:** Сервис генерации номеров, учитывающий контекст (Пространство/Тип документа).
* **Инфраструктура/Конфигурация:** Постоянное хранилище счетчиков номеров (PostgreSQL).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Генерация номера
* **Given:** Пользователь создает новый документ по шаблону с требованиями.
* **When:** Документ сохраняется.
* **Then:** Все требования в документе получают уникальный (автоматически сгенерированный) номер в формате `REQ-XXX` (где XXX — трёхзначный порядковый номер, например `REQ-001`).

### Сценарий 2: Уникальность номеров в одном пространстве
* **Given:** В одном пространстве (Space) уже существуют документы с требованиями `REQ-001`–`REQ-010`.
* **When:** Пользователь создаёт новый документ по шаблону в том же пространстве.
* **Then:** Требования нового документа получают номера, начиная с `REQ-011` — без пересечений с уже существующими.

### Сценарий 3: Независимая нумерация в разных пространствах
* **Given:** В пространстве A существуют требования `REQ-001`–`REQ-005`.
* **When:** Пользователь создаёт документ по шаблону в пространстве B.
* **Then:** Нумерация в пространстве B начинается независимо с `REQ-001`, не зависит от счётчика пространства A.
