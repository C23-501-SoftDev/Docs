# US1.1.4 Единый стиль и навигация по приложению

## Описание
Разработка базового макета (Layout) и системы шаблонов Thymeleaf, обеспечивающих единый внешний вид, навигационную структуру и стилизацию всех страниц веб-приложения «База знаний».

## Технические требования
* **Стек/Инструменты:** Thymeleaf Layout Dialect (или стандартные фрагменты), CSS3 (Flexbox/Grid), HTML5.
* **Архитектурные аспекты:**
  * Реализация мастер-страницы (`layout.html`), которая включает:
    * **Шапку (Header):** Логотип системы, имя текущего пользователя (из `Authentication`), кнопку выхода (Logout).
    * **Боковую панель (Sidebar):** Заглушка для дерева пространств и документов (см. US3.2.1).
    * **Центральную область (Main Content):** `th:replace` для динамического контента страниц.
  * Поддержка динамического отображения элементов в зависимости от роли пользователя (см. `070.role-matrix.md`):
    * Кнопка «Создать документ» видна только `Editor` и `Admin`.
    * Ссылка «Панель администратора» видна только `Admin`.
  * Реализация выхода из системы (Logout):
    * HTML-форма с методом `POST` на `/logout`.
    * Автоматическое добавление CSRF-токена в скрытое поле (`th:action`).
* **Инфраструктура/Конфигурация:**
  * Настройка `WebMvcConfigurer` для обслуживания статических ресурсов (CSS, JS, шрифты) из `/static`.
  * Настройка стилей в соответствии с концепцией **Desktop-first** (оптимизация под экраны от 1280px).
  * Единая цветовая схема (светлый фон, контрастный текст) для комфортного чтения документации.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Отображение шапки для авторизованного пользователя
* **Given:** Пользователь успешно вошел в систему (есть Cookie `JWT`).
* **When:** Пользователь переходит на любую страницу приложения (например, `/documents`).
* **Then:**
  1. В верхней части страницы отображается шапка с логотипом «База знаний».
  2. Справа в шапке отображается логин текущего пользователя.
  3. Рядом с логином отображается кнопка «Выйти».

### Сценарий 2: Разграничение видимости элементов (RBAC в UI)
* **Given:** Пользователь с ролью `Reader` авторизован.
* **When:** Просматривается главная страница.
* **Then:** Кнопка «Создать документ» **не отображается**.
* **Given:** Пользователь с ролью `Editor` авторизован.
* **When:** Просматривается главная страница.
* **Then:** Кнопка «Создать документ» **отображается**.

### Сценарий 3: Работа Logout через форму
* **Given:** Пользователь авторизован и находится на любой странице.
* **When:** Пользователь нажимает кнопку «Выйти» в шапке.
* **Then:**
  1. Отправляется POST-запрос на `/logout` с CSRF-токеном.
  2. Сервер очищает Cookie `JWT`.
  3. Пользователь перенаправляется на страницу логина.

### Сценарий 4: Адаптивность верстки (Desktop-first)
* **Given:** Окно браузера имеет ширину 1366px.
* **When:** Загружена страница.
* **Then:**
  1. Боковая панель (Sidebar) занимает ~250-300px и прижата к левому краю.
  2. Основной контент занимает оставшееся пространство.
  3. Элементы интерфейса не налезают друг на друга, присутствуют отступы.
