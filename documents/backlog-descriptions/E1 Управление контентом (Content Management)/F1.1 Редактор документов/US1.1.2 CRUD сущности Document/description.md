# US1.1.2 CRUD сущности Document

## Описание
Реализация базовых операций CRUD (Create, Read, Update, Delete) для сущности «Документ», определяющей структуру и содержимое контента в Базе знаний.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend API), PostgreSQL 18.
* **Архитектурные аспекты:** ORM Entities, Service layer для работы с Git (сохранение контента).
* **Инфраструктура/Конфигурация:** Соответствие жизненному циклу (статусы Draft/Published).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Создание документа
* **Given:** Пользователь с ролью Editor.
* **When:** Создает новый документ, заполняет атрибуты и сохраняет.
* **Then:** Документ создается в БД, контент сохраняется в репозиторий Git, и статус по умолчанию Draft.

### Сценарий 2: Чтение данных
* **Given:** Документ существует.
* **When:** Запрашивается GET-запрос к API документа.
* **Then:** Возвращаются все метаданные документа и его содержимое.
