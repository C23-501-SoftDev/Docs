# US1.1.3 Валидация входящих данных при сохранении

## Описание
Реализация валидации данных на стороне бэкенда для обеспечения целостности и корректности сохраняемого контента документа.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Hibernate Validation/Bean Validation).
* **Архитектурные аспекты:** Валидация DTO (название не пустое, допустимые символы, ограничение длины).
* **Инфраструктура/Конфигурация:** Обработка ошибок валидации и возврат соответствующих кодов ответов (400 Bad Request).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Валидация полей
* **Given:** Пользователь отправляет запрос на сохранение с некорректными данными (например, пустое название).
* **When:** Бэкенд выполняет валидацию.
* **Then:** Запрос отклоняется, система возвращает описание ошибки.

### Сценарий 2: Успешная валидация
* **Given:** Пользователь отправляет запрос с корректно заполненными данными.
* **When:** Бэкенд выполняет валидацию.
* **Then:** Валидация пройдена, система сохраняет данные (оркестрация БД + Git).
