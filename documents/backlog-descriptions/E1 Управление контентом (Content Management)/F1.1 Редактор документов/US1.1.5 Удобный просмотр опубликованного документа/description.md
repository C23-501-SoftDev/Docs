# US1.1.5 Удобный просмотр опубликованного документа

## Описание
Создание серверного шаблона Thymeleaf для страницы просмотра конкретного документа. Страница должна отображать метаданные (автор, дата), контент, отрендеренный из Wiki-разметки в HTML, и панель действий (Редактировать, История, Экспорт) в зависимости от прав пользователя.

## Технические требования
* **Стек/Инструменты:** Thymeleaf, FlexMark (или аналогичный Java Markdown Parser), CSS.
* **Архитектурные аспекты:**
  * Использование общего `layout.html`.
  * **Серверный рендеринг:** Контроллер получает `Document` из БД, преобразует поле `content` (Markdown) в безопасный HTML с помощью библиотеки-парсера, добавляет результат в `Model`.
  * Безопасность: Предотвращение XSS-атак при рендеринге пользовательского Markdown (экранирование HTML-тегов, санитизация вывода).
  * **Динамические действия (UI):**
    * Кнопка «Редактировать»: отображается, если у пользователя есть право `WRITE` на документ или пространство.
    * Кнопка «История версий»: отображается всем авторизованным.
    * Кнопка «Экспорт»: отображается всем авторизованным.
    * Кнопка «Удалить»: отображается, если есть права `WRITE`.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Просмотр документа Читателем
* **Given:** Пользователь с ролью `Reader` открыл опубликованный документ.
* **When:** Страница загружена.
* **Then:**
  1. В заголовке страницы отображается `title` документа.
  2. Под заголовком строка: «Автор: Ivanov», «Обновлено: 01.01.2026».
  3. Содержимое отображается в виде форматированного текста (заголовки, списки, код).
  4. Кнопка **«Редактировать» отсутствует**.
  5. Кнопки «История» и «Экспорт» **присутствуют**.

### Сценарий 2: Просмотр документа Редактором (Автором)
* **Given:** Пользователь с ролью `Editor` (и он же автор) открыл документ.
* **When:** Страница загружена.
* **Then:**
  1. Отображается плашка со статусом (`Draft` или `Published`).
  2. Кнопка **«Редактировать» отображается** и ведет на `/documents/{id}/edit`.
  3. Кнопка **«Удалить» отображается**.

### Сценарий 3: Отображение вложений
* **Given:** К документу прикреплены файлы (реализовано в US1.3.1).
* **When:** Открыта страница просмотра.
* **Then:**
  1. Под основным текстом отображается блок «Вложения».
  2. Отображается список файлов с иконками типов и возможностью скачать.

### Сценарий 4: Навигация по иерархии (Хлебные крошки)
* **Given:** Документ находится в пространстве «Проект Альфа» и вложен в документ «Спецификация».
* **When:** Открыта страница просмотра.
* **Then:** В верхней части контента отображаются навигационные крошки: `Главная > Проект Альфа > Спецификация > Текущий документ`.
