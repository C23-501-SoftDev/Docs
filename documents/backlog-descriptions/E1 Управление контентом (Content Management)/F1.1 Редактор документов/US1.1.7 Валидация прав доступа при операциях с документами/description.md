# US1.1.7 Валидация прав доступа при операциях с документами

## Описание
Реализация проверки полномочий пользователя при создании, редактировании, перемещении и удалении документов на основе RBAC и прав доступа к Пространствам.

## Технические требования
* **Стек/Инструменты:** Spring Security, RBAC Module.
* **Архитектурные аспекты:**
    * Проверка прав `WRITE` или `OWNER` на уровне Пространства (Space) перед сохранением/удалением документа.
    * Проверка прав доступа при изменении родительского документа (необходимы права на запись в целевом родителе).
    * Интеграция с сервисом авторизации для получения текущих ролей пользователя.
* **Инфраструктура/Конфигурация:** Возврат кода 403 Forbidden при недостаточности прав.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Попытка изменения без прав на Space
* **Given:** Пользователь имеет права только на чтение (`READ`) в Пространстве.
* **When:** Пользователь пытается сохранить изменения в документе.
* **Then:** Система возвращает 403 Forbidden, изменения не сохраняются ни в БД, ни в Git.

### Сценарий 2: Перемещение документа между пространствами
* **Given:** Пользователь имеет права на запись в исходном Пространстве, но не имеет прав в целевом.
* **When:** Пользователь пытается изменить `spaceId` документа.
* **Then:** Система блокирует операцию с кодом 403.
