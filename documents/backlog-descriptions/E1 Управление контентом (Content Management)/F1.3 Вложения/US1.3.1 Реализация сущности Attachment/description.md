# US1.3.1 Реализация сущности Attachment

## Описание
Реализация поддержки прикрепляемых файлов (вложений) к документам, включая загрузку через drag-and-drop или выбор файла, хранение (на SMB-шаре) и удаление.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot, SMB API (интеграция с файловым хранилищем).
* **Архитектурные аспекты:** Сущность `Attachment` (метаданные в БД, ссылка на путь к файлу на SMB).
* **Инфраструктура/Конфигурация:** Права доступа сервисного аккаунта к SMB шаре. Потоковая загрузка/отдача файлов (Multipart).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Успешная загрузка
* **Given:** Редактируемый документ.
* **When:** Редактор прикрепляет файл через drag-and-drop или кнопку выбора файла.
* **Then:** Файл сохраняется в целевой SMB-директории, метаданные записываются в БД и связываются с документом.

### Сценарий 2: Удаление вложения
* **Given:** К документу прикреплён файл.
* **When:** Редактор удаляет вложение.
* **Then:** Файл удаляется с SMB-шары, запись метаданных удаляется из БД, вложение больше не отображается в документе.

### Сценарий 3: Превышение допустимого размера файла
* **Given:** Редактируемый документ.
* **When:** Пользователь пытается загрузить файл, превышающий допустимый размер.
* **Then:** Система отклоняет загрузку и отображает понятное сообщение об ошибке с указанием максимально допустимого размера.

### Сценарий 4: Недопустимый тип файла
* **Given:** Редактируемый документ.
* **When:** Пользователь пытается загрузить файл с запрещённым расширением.
* **Then:** Система отклоняет загрузку и отображает сообщение с перечнем допустимых типов файлов.
