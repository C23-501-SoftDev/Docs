# US3.1.5 Поиск по пространствам

## Описание
Реализация функционала поиска рабочих пространств в системе, позволяющего быстро переключаться между ними или находить нужное пространство по его названию.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot, PostgreSQL 18.
* **Архитектурные аспекты:** Добавление эндпоинта поиска по сущности `Space`, обновление компонента навигации (Sidebar).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Успешный поиск пространства
* **Given:** В системе созданы рабочие пространства.
* **When:** Пользователь вводит название пространства в поле поиска.
* **Then:** Система отображает список пространств, соответствующих введенному запросу.

### Сценарий 2: Поиск без результатов
* **Given:** Пространство с введенным названием отсутствует.
* **When:** Выполняется поиск.
* **Then:** Система выводит сообщение "Пространства не найдены".

### Сценарий 3: Учёт прав доступа
* **Given:** Пользователь имеет доступ к определенному набору пространств.
* **When:** Выполняется поиск по пространствам.
* **Then:** В результатах отображаются только те пространства, к которым у пользователя есть права на просмотр.

### Сценарий 4: Регистронезависимый поиск и частичные совпадения
* **Given:** Существует пространство "Отдел Маркетинга".
* **When:** Пользователь ищет "маркетинг".
* **Then:** Система возвращает найденное пространство (поиск case-insensitive).

### Технические критерии готовности (Definition of Done)
* **Логирование:** Ошибки поиска или аномально долгие запросы (более 500мс) логируются на сервере с указанием ID пользователя.
* **Безопасность:** Использование `PreparedStatement` для защиты от SQL-инъекций; фильтрация результатов на уровне доступа (RBAC).
* **API Документация:** Эндпоинт поиска соответствует Swagger, содержит описание параметров запроса и возможных кодов ответов (200, 400, 403).
