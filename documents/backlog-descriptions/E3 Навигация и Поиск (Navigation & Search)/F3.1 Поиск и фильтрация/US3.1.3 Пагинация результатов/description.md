# US3.1.3 Пагинация результатов

## Описание
Реализация постраничного вывода (пагинации) результатов поиска для повышения производительности и удобства чтения пользователем.

## Технические требования
* **Стек/Инструменты:** Spring Data JPA (Pageable).
* **Архитектурные аспекты:** API должен возвращать данные небольшими порциями (страницами). Передача параметров `page` и `size` в REST запросе.
* **Инфраструктура/Конфигурация:** Включение параметров пагинации в Query-параметры API метода поиска.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Отображение страницы
* **Given:** Поиск возвращает большое количество результатов.
* **When:** Пользователь делает запрос с параметрами страницы.
* **Then:** Система возвращает строго количество записей, указанное в `size`, и общую информацию (всего страниц/записей), если требуется.

### Сценарий 2: Обработка невалидных параметров
* **Given:** Пользователь запрашивает отрицательную страницу или размер страницы, превышающий максимально допустимый (например, 1000).
* **When:** Выполняется запрос к API.
* **Then:** Система возвращает ошибку 400 Bad Request с пояснением допустимых диапазонов.

### Сценарий 3: Пустая страница
* **Given:** Пользователь запрашивает номер страницы, превышающий общее количество доступных страниц.
* **When:** Выполняется поиск.
* **Then:** Система возвращает пустой список результатов (HTTP 200) и метаданные, указывающие, что текущая страница пуста.

### Сценарий 4: Совместная работа с фильтрами
* **Given:** Пользователь выполнил поисковый запрос с применением фильтров (например, по дате или пространству).
* **When:** Пользователь переключается между страницами результатов (например, переходит на страницу 2).
* **Then:** Состояние фильтров сохраняется, и система возвращает следующую страницу результатов, отфильтрованную согласно исходным критериям.

### Технические критерии готовности (Definition of Done)
* **API Документация:** Swagger содержит описание параметров `page` (индексация с 0) и `size`, а также пример структуры ответа (Page Object: content, totalElements, totalPages, number).
* **Производительность:** Использование `Pageable` в Spring Data JPA должно сопровождаться корректной сортировкой (`Sort`), чтобы порядок элементов между страницами был детерминированным.
* **Безопасность:** Ограничение максимального значения `size` на уровне конфигурации приложения для предотвращения атак на исчерпание памяти (Memory Exhaustion).
* **Фильтрация:** Эндпоинт поиска должен поддерживать передачу параметров фильтрации (dateFrom, dateTo, spaceId и т.д.) совместно с параметрами пагинации в рамках одного REST-запроса.
