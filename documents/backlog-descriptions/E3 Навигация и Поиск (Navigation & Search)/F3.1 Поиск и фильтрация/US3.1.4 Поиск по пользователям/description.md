# US3.1.4 Поиск по пользователям

## Описание
Реализация функционала поиска зарегистрированных пользователей в системе, позволяющего администраторам или другим пользователям быстро находить коллег по ФИО или логину.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot, PostgreSQL 18.
* **Архитектурные аспекты:** Добавление эндпоинта поиска по сущности `User`, интеграция со строкой поиска или отдельной админ-страницей.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Успешный поиск пользователя
* **Given:** В системе зарегистрированы пользователи.
* **When:** Администратор вводит часть ФИО или логина в поле поиска пользователей.
* **Then:** Система отображает список пользователей, соответствующих введенному запросу.

### Сценарий 2: Поиск без результатов
* **Given:** Пользователь с введенным именем отсутствует.
* **When:** Выполняется поиск.
* **Then:** Система выводит сообщение "Пользователи не найдены".

### Сценарий 3: Масштабируемость и производительность
* **Given:** В системе зарегистрировано большое количество пользователей (более 10 000).
* **When:** Выполняется поиск по неполному имени.
* **Then:** Время ответа от сервера не превышает 300мс при использовании GIN-индексов по полям ФИО/логин.

### Сценарий 4: Безопасность данных (Masking)
* **Given:** Обычный пользователь (не администратор) выполняет поиск по сотрудникам.
* **When:** Результаты поиска возвращаются.
* **Then:** Система отображает только публичную информацию (ФИО, должность, отдел), скрывая чувствительные данные (email, номер телефона, если применимо).

### Технические критерии готовности (Definition of Done)
* **Логирование:** Аудит поисковых запросов в разделе "Пользователи" для обеспечения безопасности и предотвращения злоупотреблений.
* **Безопасность:** Использование параметризованных запросов; отсутствие возможности извлечения всех пользователей системы через подстановочные знаки (Wildcards) без ограничений по пагинации.
* **API Документация:** Соответствие эндпоинта поиска пользователей Swagger, включая описание всех параметров фильтрации и кодов ответов (200, 403).
