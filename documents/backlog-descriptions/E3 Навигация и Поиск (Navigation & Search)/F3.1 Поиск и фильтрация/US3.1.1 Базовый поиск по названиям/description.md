# US3.1.1 Базовый поиск по названиям

## Описание
Реализация базового функционала поиска документов по их заголовкам для обеспечения быстрого доступа к информации.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend API), PostgreSQL (SQL `LIKE` или индексы БД).
* **Архитектурные аспекты:** API endpoint, принимающий поисковую строку. Валидация входных данных, ограничение размера выборки.
* **Инфраструктура/Конфигурация:** Использование GIN-индексов или аналогичных средств PostgreSQL для оптимизации поиска по текстовым полям (названиям).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Успешный поиск
* **Given:** В системе существуют документы с заголовками, содержащими поисковую фразу.
* **When:** Пользователь вводит фразу в строку поиска и нажимает поиск.
* **Then:** Система возвращает список всех документов, в названии которых встречается введённая фраза.

### Сценарий 2: Отсутствие результатов
* **Given:** Фраза поиска не соответствует названиям существующих документов.
* **When:** Пользователь выполняет поиск.
* **Then:** Система возвращает пустой список результатов с сообщением "Документы не найдены".

### Сценарий 3: Учёт прав доступа
* **Given:** В системе существуют документы, содержащие поисковую фразу, но часть из них находится в пространствах, недоступных текущему пользователю.
* **When:** Пользователь выполняет поиск.
* **Then:** В результатах отображаются только документы из пространств, к которым у пользователя есть права доступа. Закрытые документы не появляются в выдаче.
