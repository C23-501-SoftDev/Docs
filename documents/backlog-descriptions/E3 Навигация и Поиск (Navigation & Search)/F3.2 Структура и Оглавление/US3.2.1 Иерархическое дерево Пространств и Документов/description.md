# US3.2.1 Иерархическое дерево Пространств и Документов

## Описание
Реализация иерархического оглавления (навигационного дерева) для отображения структуры пространств и документов внутри них.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot, PostgreSQL, Frontend (Tree UI).
* **Архитектурные аспекты:** Хранение иерархии (родитель-потомок) в БД. Рекурсивный запрос или Closure Table для сбора дерева документов.
* **Инфраструктура/Конфигурация:** Обеспечение консистентности дерева при создании/перемещении/удалении документов.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Просмотр дерева
* **Given:** Пользователь находится на главной странице или в пространстве.
* **When:** Загружается навигационная панель.
* **Then:** Система отображает иерархическую структуру (пространства -> папки/документы).
