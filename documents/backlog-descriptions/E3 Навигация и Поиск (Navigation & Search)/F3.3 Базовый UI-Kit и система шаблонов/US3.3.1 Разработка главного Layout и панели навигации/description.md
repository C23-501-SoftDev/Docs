# US3.3.1 Разработка главного Layout и панели навигации

## Описание
Создание базовой мастер-страницы (Layout) Thymeleaf, которая определяет единую структуру HTML-документа, шапку (Header) с информацией о пользователе и кнопкой выхода, а также область для бокового меню и основного контента. Этот Layout будет использоваться всеми остальными страницами приложения для обеспечения визуальной консистентности.

## Технические требования
* **Стек/Инструменты:** Thymeleaf, HTML5, CSS3 (Flexbox/Grid), Spring MVC.
* **Архитектурные аспекты:**
  * Создание файла `layout.html` в директории `src/main/resources/templates/fragments/`.
  * Структура Layout:
    * **`<head>`**: Подключение общих CSS-стилей, шрифтов, мета-тегов (Viewport для Desktop).
    * **`<body>`**: Трехколоночная или двухколоночная структура (Flexbox):
      1. **Header (шапка)**: Фиксированная сверху или статичная. Содержит логотип «База знаний», имя текущего пользователя (`${#authentication.name}`), кнопку «Выйти» (форма POST на `/logout` с CSRF).
      2. **Sidebar Container**: Блок для левой панели (`<div class="sidebar">`). Содержимое будет заполняться дочерними шаблонами через `th:replace` или `th:insert` (в частности, для US3.3.2).
      3. **Main Container**: Блок для основного контента (`<div class="content">`), куда вставляется контент конкретных страниц через `th:replace`.
  * **Logout:** Реализован как HTML-форма (`method="post"`, `th:action="@{/logout}"`) со скрытым полем CSRF.
  * **Динамическое отображение элементов:**
    * Использование Spring Security Dialect для Thymeleaf (`sec:authorize`).
    * Ссылка на «Админ-панель» (`/admin`) отображается только при наличии роли `ROLE_ADMIN`.
    * Кнопка «Создать документ» отображается при ролях `ROLE_EDITOR` или `ROLE_ADMIN`.
* **Инфраструктура/Конфигурация:**
  * Настройка `WebMvcConfigurer` для прямого доступа к ресурсам `/css/**`, `/js/**`, `/images/**`.
  * Подключение библиотеки `thymeleaf-extras-springsecurity6` для интеграции с Spring Security в шаблонах.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Отображение Layout для авторизованного пользователя
* **Given:** Пользователь авторизован (есть валидная Cookie `JWT`).
* **When:** Открыта любая страница приложения (например, `/documents`).
* **Then:**
  1. Вверху страницы видна шапка с названием системы и логином пользователя.
  2. Рядом с именем отображается кнопка/иконка «Выход».
  3. В левой части экрана зарезервировано место для боковой панели (даже если она пустая).

### Сценарий 2: RBAC в интерфейсе (Видимость кнопок)
* **Given:** Пользователь с ролью `Reader`.
* **When:** Загружена страница.
* **Then:** Кнопка «Создать» в интерфейсе **отсутствует**.
* **Given:** Пользователь с ролью `Editor`.
* **When:** Загружена страница.
* **Then:** Кнопка «Создать» в интерфейсе **отображается**.

### Сценарий 3: Выход из системы (Logout)
* **Given:** Пользователь находится на странице с Layout.
* **When:** Нажата кнопка «Выйти».
* **Then:**
  1. Отправляется POST-запрос на `/logout`.
  2. Cookie `JWT` и `XSRF-TOKEN` очищаются сервером.
  3. Пользователь перенаправляется на страницу входа (`/login`).

### Сценарий 4: Наследование шаблона
* **Given:** Разработчик создает новую страницу `document-view.html`.
* **When:** Страница рендерится.
* **Then:** Страница автоматически наследует шапку и структуру из `layout.html`, разработчику нужно только заполнить блок `main-content`.
