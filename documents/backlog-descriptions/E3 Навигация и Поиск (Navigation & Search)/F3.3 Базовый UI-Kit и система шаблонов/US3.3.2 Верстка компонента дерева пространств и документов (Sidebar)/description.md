# US3.3.2 Верстка компонента дерева пространств и документов (Sidebar)

## Описание
Разработка визуального компонента боковой панели (Sidebar) с использованием HTML и CSS, который отображает иерархическое оглавление: список пространств (`Spaces`) и вложенных в них документов. На данном этапе верстка статическая (данные замоканы), интерактивность и подгрузка данных реализуются в других задачах (US3.2.1).

## Технические требования
* **Стек/Инструменты:** HTML5, CSS3 (псевдоклассы, каскады), Thymeleaf Fragments.
* **Архитектурные аспекты:**
  * Компонент разрабатывается как отдельный фрагмент `fragments/sidebar.html`.
  * Визуальная структура:
    * Заголовок панели: «Оглавление» или «Навигация».
    * Скроллируемая область (`overflow-y: auto`).
    * **Верстка дерева:**
      * Реализуется через вложенные списки `<ul>` и `<li>`.
      * Для отступов вложенных элементов используется `padding-left` или `margin-left`.
      * Стилизация активного/текущего документа.
      * Иконки для типов элементов: 📁 для Пространства, 📄 для Документа (на чистом CSS или эмодзи).
  * **Внешний вид:**
    * Фон панели: нейтральный серый (например, `#f5f5f5`).
    * Ширина: фиксированная (например, `280px`).
    * Высота: занимает всю доступную высоту под шапкой.
  * **Мок-данные:** Для демонстрации верстки в шаблон вставляется статический пример дерева:
    * `📁 Проект Альфа`
      * `📄 Архитектура системы`
      * `📄 Требования`
    * `📁 Разработка`
      * `📄 Coding Style Guide`

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Отображение статического дерева
* **Given:** Открыта страница с подключенным фрагментом `sidebar.html`.
* **When:** Страница загружена.
* **Then:**
  1. В левой части экрана отображается панель шириной не более 300px.
  2. В панели видны названия пространств и документов с иерархическими отступами.

### Сценарий 2: Визуальное выделение уровней
* **Given:** Отображается статическое дерево.
* **When:** Пользователь смотрит на панель.
* **Then:**
  1. Пространства визуально отделены от документов (например, жирным шрифтом или иконкой папки).
  2. Документы внутри пространства имеют отступ слева ~16-20px.
  3. Документы третьего уровня (вложенные в другие документы) имеют дополнительный отступ.

### Сценарий 3: Скроллирование
* **Given:** Количество мок-элементов превышает высоту экрана (более 20 пунктов).
* **When:** Страница загружена.
* **Then:** Появляется вертикальная полоса прокрутки только внутри боковой панели, позволяющая промотать список, в то время как шапка и основная область остаются на месте.

### Сценарий 4: Адаптация к Layout
* **Given:** Страница сверстана с использованием `layout.html`.
* **When:** Фрагмент `sidebar.html` вставлен в левую колонку.
* **Then:** Не происходит смещения основного контента, панель и контент располагаются рядом по горизонтали (Flexbox/Grid).
