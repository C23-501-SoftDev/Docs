# US2.1.1 Реализация статусов Draft и Published

## Описание
Реализация жизненного цикла документа с поддержкой статусов "Черновик" (Draft) и "Опубликован" (Published) для обеспечения контроля публикации контента.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend), PostgreSQL 18 (DB).
* **Архитектурные аспекты:** Базовая логика обработки статусов в бизнес-слое приложения, валидация прав при переходе между статусами.
* **Инфраструктура/Конфигурация:** Обновление схемы БД (таблица документов с полем `status`).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Создание документа
* **Given:** Пользователь авторизован как Редактор или имеет право WRITE на пространство.
* **When:** Пользователь создает новый документ.
* **Then:** Документ создается с заданным статусом (по умолчанию "Draft") и возможностью редактирования.

### Сценарий 2: Публикация документа
* **Given:** Документ находится в статусе "Черновик".
* **When:** Автор сохраняет/публикует готовый документ.
* **Then:** Статус меняется на "Опубликован" (Published), документ остается доступен для чтения всем пользователям, имеющим права доступа к соответствующему пространству.
