# US2.1.2 Soft-удаление и восстановление

## Описание
Реализация процесса удаления документов через пометку "удален" (Soft-delete), позволяющего скрывать документы из поисковой выдачи, но сохранять данные для возможного восстановления.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend), PostgreSQL 18.
* **Архитектурные аспекты:** Реализация логического флага `is_deleted` (или поле `status`). Обновление API запросов поиска (исключение удаленных).
* **Инфраструктура/Конфигурация:** Настройка прав доступа для выполнения Soft-delete и операций восстановления в "Админ-корзине".

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Soft-delete
* **Given:** Документ находится в статусе "Опубликован".
* **When:** Редактор выполняет операцию удаления.
* **Then:** Документ переходит в статус "Удален", исчезает из общего поиска и списка документов.

### Сценарий 2: Восстановление из Корзины
* **Given:** Документ находится в статусе "Удален".
* **When:** Администратор выполняет операцию восстановления через "Корзину".
* **Then:** Документ меняет статус на "Опубликован" и снова становится доступен обычным пользователям.
