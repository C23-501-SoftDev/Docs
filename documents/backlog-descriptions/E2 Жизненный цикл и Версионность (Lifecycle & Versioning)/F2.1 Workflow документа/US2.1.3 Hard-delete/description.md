# US2.1.3 Hard-delete

## Описание
Реализация процесса окончательного физического удаления документа из системы (Hard-delete).

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend), Git Server-side, PostgreSQL 18.
* **Архитектурные аспекты:** Удаление всех связанных записей из БД и физических файлов из репозитория Git/файловой системы. Ограничение доступа только для администратора.
* **Инфраструктура/Конфигурация:** Выполнение операции удаления данных в транзакции (БД + репозиторий).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Физическое удаление
* **Given:** Документ находится в корзине ("Удален").
* **When:** Администратор выполняет операцию Hard-delete.
* **Then:** Запись о документе удаляется из БД, а связанные файлы/версии удаляются из Git-репозитория и файловой системы.
