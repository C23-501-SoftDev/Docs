# US2.2.1 Git commit при сохранении

## Описание
Автоматическая фиксация изменений каждой версии документа в Git-репозитории при сохранении (коммит) для обеспечения версионности.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend), Git (API integration), PostgreSQL 18.
* **Архитектурные аспекты:** Интеграция службы Git на стороне сервера. При каждом сохранении документа создается commit в репозитории и записывается метаданные (автор, дата) в базу данных.
* **Инфраструктура/Конфигурация:** Доступ сервера к локальному Git-репозиторию (созданному в US4.1.1).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Фиксация версии
* **Given:** Документ был изменен (содержимое или атрибуты).
* **When:** Редактор выполняет сохранение документа.
* **Then:** В Git-репозитории создается новый коммит, связанный с этим документом, а в БД фиксируются метаданные версии (хэш, автор, время).
