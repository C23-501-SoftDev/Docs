# US2.2.4 Безопасный откат версии

## Описание
Функциональность восстановления документа (откат изменений) к состоянию любой выбранной исторической версии.

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend), Git.
* **Архитектурные аспекты:** Реализация отката через команду Git revert или checkout нужного коммита. Обязательный перезапуск индексации в БД после отката данных файла.
* **Инфраструктура/Конфигурация:** Транзакционность операции (файловая система + БД).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Откат версии
* **Given:** Редактор просматривает историю версий документа.
* **When:** Редактор выбирает версию для восстановления и подтверждает действие.
* **Then:** Система восстанавливает содержимое файла из Git-репозитория до состояния выбранного коммита, сохраняет эту "старую" версию как новую текущую (для сохранения целостности истории) и делает ее активной в БД.
