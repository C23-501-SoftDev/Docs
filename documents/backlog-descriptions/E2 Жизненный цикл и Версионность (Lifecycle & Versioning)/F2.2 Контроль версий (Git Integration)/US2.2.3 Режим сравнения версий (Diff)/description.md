# US2.2.3 Режим сравнения версий (Diff)

## Описание
Визуализация изменений между двумя версиями документа (предыдущей и текущей или выбранной любой парой).

## Технические требования
* **Стек/Инструменты:** Java/Spring Boot (Backend Diff logic), HTML (Diff view).
* **Архитектурные аспекты:** Генерация визуального сравнения (Diff) в стиле GitHub (удаленные/добавленные строки) через Git-команды на сервере.
* **Инфраструктура/Конфигурация:** Обработка данных Git diff на бэкенде и отправка на клиента для отрисовки.

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Сравнение версий
* **Given:** Пользователь выбрал две версии документа в истории.
* **When:** Активируется режим сравнения.
* **Then:** Система отображает визуальное представление изменений (подсвеченные добавленные и удаленные фрагменты текста).
