# US2.2.7 Обеспечение транзакционности обновления БД и Git

## Описание
Реализация механизма согласованного обновления PostgreSQL и Git для предотвращения рассинхронизации данных при сбоях.

## Технические требования
* **Logic:** Использование механизма "Write-Ahead Logging" или двухфазной фиксации, обеспечивающей, что если Git commit не удался — запись в БД откатывается (или наоборот).

## Критерии приемки (Acceptance Criteria)

### Сценарий 1: Откат при ошибке Git
* **Given:** Редактор сохраняет документ.
* **When:** Сбой происходит при записи в Git-репозиторий (например, нет места на диске), а БД попытку записи еще не завершила.
* **Then:** Система откатывает изменения в БД, чтобы метаданные не указывали на несуществующий коммит.
