# Глоссарий предметной области (Ubiquitous Language)

| Термин | Определение |
| :--- | :--- |
| **База знаний (БЗ)** | Веб-приложение для централизованного хранения и управления технической и аналитической документацией. |
| **Пространство (Space)** | Логическая группировка документов (по проектам, отделам или темам) с едиными настройками прав доступа. |
| **Документ** | Основная единица хранения информации, созданная на основе шаблона и содержащая текст в формате Wiki-разметки. |
| **Wiki-разметка** | Облегченный язык разметки (Markdown), используемый для оформления текста в документах. |
| **Шаблон документа** | Предустановленная структура документа с набором специфических атрибутов, зависящая от роли пользователя (Разработчик, Аналитик, Администратор). |
| **Версия документа** | Состояние документа в определенный момент времени, зафиксированное в системе контроля версий (Git). |
| **Git** | Распределенная система контроля версий, используемая в БЗ для хранения истории изменений документов и обеспечения их версионности. |
| **CRUD** | (Create, Read, Update, Delete) Базовые операции работы с данными: создание, чтение, обновление и удаление. |
| **UI (User Interface)** | Пользовательский интерфейс, через который осуществляется взаимодействие с Базой знаний. |
| **Черновик (Draft)** | Начальный статус документа, видимый только автору и администраторам. |
| **Опубликован (Published)** | Статус документа, при котором он доступен для чтения пользователям с соответствующими правами. |
| **Soft-delete (Мягкое удаление)** | Пометка документа как удаленного без физического стирания из базы данных (перемещение в "Корзину"). |
| **Hard-delete (Физическое удаление)** | Полное удаление данных документа из системы (БД и Git). |
| **Роль** | Набор прав, назначенных пользователю. Глобальные роли: `GUEST` (гость, без привилегий), `READER` (читатель), `EDITOR` (редактор). Дополнительно пользователь может иметь флаг администратора (`is_admin`). |
| **GUEST** | Базовая роль пользователя. Не имеет глобальных привилегий, кроме возможности входа в систему. Все права на пространства и документы получает через явное назначение администратором или владельцем. |
| **READER** | Роль, дающая глобальное право на чтение любых документов в системе. |
| **EDITOR** | Роль, дающая глобальное право на создание, редактирование и удаление любых документов в системе. |
| **ADMIN (Администратор)** | Пользователь с флагом `is_admin = true`. Имеет доступ к панели администратора, может управлять пользователями, пространствами и правами доступа. Не имеет автоматических прав на контент — они определяются ролью (`GUEST`/`READER`/`EDITOR`). |
| **Группа пользователей** | Именованное объединение существующих пользователей, создаваемое администратором. Используется для массового назначения прав на пространства. Пользователь может входить в несколько групп одновременно. Права, полученные через группу, суммируются с личными правами пользователя (действует максимальное из доступных прав). |
| **Вложение** | Файл, прикрепленный к документу и хранящийся вместе с ним. |
| **Оглавление** | Иерархическая структура, отображающая взаимосвязи и вложенность документов внутри пространства. |
| **Экспорт** | Процесс преобразования документа в форматы PDF, DOCX или HTML для внешнего использования. |
| **RESTful API** | Архитектурный стиль взаимодействия компонентов системы через протокол HTTP, используемый для интеграции UI и серверной части. |
