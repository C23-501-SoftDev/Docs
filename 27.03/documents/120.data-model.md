# Моделирование данных (Data Modeling)

## ER-диаграмма (Логическая модель)

```mermaid
erDiagram
    USER ||--o{ DOCUMENT : "создает"
    USER ||--o{ SPACE : "управляет"
    SPACE ||--o{ DOCUMENT : "содержит"
    DOCUMENT ||--o{ ATTACHMENT : "имеет"
    DOCUMENT ||--o{ VERSION : "имеет историю"
    
    USER {
        long id PK
        string login
        string password_hash
        string email
        string role "Admin, Editor, Reader"
    }

    SPACE {
        long id PK
        string name
        string description
        long owner_id FK
    }

    DOCUMENT {
        long id PK
        string title
        text content
        string status "Draft, Published, Deleted"
        long author_id FK
        long space_id FK
        datetime created_at
        datetime updated_at
        boolean is_deleted
    }

    VERSION {
        long id PK
        long document_id FK
        string git_hash
        long author_id FK
        datetime created_at
        text diff
    }

    ATTACHMENT {
        long id PK
        long document_id FK
        string filename
        string content_type
        long size
        string storage_path
    }
```

## Описание таблиц

### 1. Users
Хранит учетные данные и глобальные роли пользователей.

### 2. Spaces
Определяет логические группы документов. Связана с пользователем-владельцем (ответственным редактором).

### 3. Documents
Основная таблица контента. Поле `status` управляет видимостью, а `is_deleted` используется для Soft-удаления.

### 4. Versions
Служит для связи метаданных документа с физическими коммитами в Git-репозитории. Хранит ссылку на автора правки.

### 5. Attachments
Метаданные файлов. Сами файлы хранятся в файловой системе или объектном хранилище, в БД только путь.
