# OPSX: Архивация (Archive)

Архивация завершенного изменения в экспериментальном рабочем процессе

Архивация завершенного изменения в экспериментальном рабочем процессе.

**Ввод**: Опционально укажите имя изменения после `/opsx:archive` (например, `/opsx:archive add-auth`). Если оно пропущено, проверьте, можно ли его определить из контекста беседы. Если контекст неясен или неоднозначен, вы ДОЛЖНЫ запросить список доступных изменений.

**Шаги**

1. **Если имя изменения не указано, предложите выбор**

   Выполните `openspec list --json`, чтобы получить список доступных изменений. Используйте инструмент **AskUserQuestion**, чтобы пользователь мог выбрать.

   Показывайте только активные изменения (еще не архивированные).
   Включите используемую схему для каждого изменения, если она доступна.

   **ВАЖНО**: НЕ гадайте и не выбирайте изменение автоматически. Всегда давайте пользователю возможность выбора.

2. **Проверка статуса готовности артефактов**

   Выполните `openspec status --change "<name>" --json`, чтобы проверить готовность артефактов.

   Разберите JSON, чтобы понять:
   - `schemaName`: Используемый рабочий процесс.
   - `artifacts`: Список артефактов с их статусом (`done` или другой).

   **Если какие-либо артефакты не в статусе `done`:**
   - Отобразите предупреждение со списком незавершенных артефактов.
   - Запросите подтверждение пользователя для продолжения.
   - Продолжайте, если пользователь подтвердит.

3. **Проверка статуса выполнения задач**

   Прочитайте файл задач (обычно `tasks.md`), чтобы проверить наличие незавершенных задач.

   Подсчитайте задачи, отмеченные `- [ ]` (незавершенные) и `- [x]` (завершенные).

   **Если найдены незавершенные задачи:**
   - Отобразите предупреждение с количеством незавершенных задач.
   - Запросите подтверждение пользователя для продолжения.
   - Продолжайте, если пользователь подтвердит.

   **Если файл задач отсутствует:** Продолжайте без предупреждения о задачах.

4. **Оценка состояния синхронизации дельта-спецификаций**

   Проверьте наличие дельта-спецификаций в `openspec/changes/<name>/specs/`. Если их нет, продолжайте без запроса синхронизации.

   **Если дельта-спецификации существуют:**
   - Сравните каждую дельта-спецификацию с соответствующей основной спецификацией в `openspec/specs/<capability>/spec.md`.
   - Определите, какие изменения будут применены (добавления, изменения, удаления, переименования).
   - Покажите сводное резюме перед запросом.

   **Варианты выбора:**
   - Если требуются изменения: "Синхронизировать сейчас (рекомендуется)", "Архивировать без синхронизации".
   - Если уже синхронизировано: "Архивировать сейчас", "Все равно синхронизировать", "Отмена".

   Если пользователь выбирает синхронизацию, используйте инструмент Task (subagent_type: "general-purpose", prompt: "Используй инструмент Skill для вызова openspec-sync-specs для изменения '<name>'. Анализ дельта-спецификаций: <включите проанализированное резюме дельта-спецификаций>"). Переходите к архивации независимо от выбора.

5. **Выполнение архивации**

   Создайте директорию архива, если она не существует:
   ```bash
   mkdir -p openspec/changes/archive
   ```

   Сгенерируйте целевое имя, используя текущую дату: `YYYY-MM-DD-<change-name>`

   **Проверьте, существует ли уже цель:**
   - Если да: Завершите с ошибкой, предложите переименовать существующий архив или использовать другую дату.
   - Если нет: Переместите директорию изменения в архив.

   ```bash
   mv openspec/changes/<name> openspec/changes/archive/YYYY-MM-DD-<name>
   ```

6. **Отображение резюме**

   Покажите резюме завершения архивации, включая:
   - Имя изменения.
   - Использованную схему.
   - Местоположение архива.
   - Статус синхронизации спецификаций (синхронизировано / синхронизация пропущена / нет дельта-спецификаций).
   - Примечание о любых предупреждениях (незавершенные артефакты/задачи).

**Вывод при успехе**

```
## Архивация завершена

**Изменение:** <change-name>
**Схема:** <schema-name>
**Архивировано в:** openspec/changes/archive/YYYY-MM-DD-<name>/
**Спецификации:** ✓ Синхронизированы с основными спецификациями

Все артефакты готовы. Все задачи выполнены.
```

**Вывод при успехе (нет дельта-спецификаций)**

```
## Архивация завершена

**Изменение:** <change-name>
**Схема:** <schema-name>
**Архивировано в:** openspec/changes/archive/YYYY-MM-DD-<name>/
**Спецификации:** Нет дельта-спецификаций

Все артефакты готовы. Все задачи выполнены.
```

**Вывод при успехе с предупреждениями**

```
## Архивация завершена (с предупреждениями)

**Изменение:** <change-name>
**Схема:** <schema-name>
**Архивировано в:** openspec/changes/archive/YYYY-MM-DD-<name>/
**Спецификации:** Синхронизация пропущена (по выбору пользователя)

**Предупреждения:**
- Архивировано с 2 незавершенными артефактами
- Архивировано с 3 незавершенными задачами
- Синхронизация дельта-спецификаций была пропущена (по выбору пользователя)

Проверьте архив, если это не было преднамеренным.
```

**Вывод при ошибке (архив уже существует)**

```
## Архивация не удалась

**Изменение:** <change-name>
**Цель:** openspec/changes/archive/YYYY-MM-DD-<name>/

Целевая директория архива уже существует.

**Варианты:**
1. Переименовать существующий архив
2. Удалить существующий архив, если это дубликат
3. Подождать другой даты для архивации
```

**Ограничения (Guardrails)**

- Всегда запрашивайте выбор изменения, если оно не указано.
- Используйте граф артефактов (openspec status --json) для проверки готовности.
- Не блокируйте архивацию из-за предупреждений — просто информируйте и подтверждайте.
- Сохраняйте `.openspec.yaml` при перемещении в архив (он перемещается вместе с директорией).
- Показывайте четкое резюме того, что произошло.
- Если запрошена синхронизация, используйте инструмент Skill для вызова `openspec-sync-specs` (управляется агентом).
- Если дельта-спецификации существуют, всегда проводите оценку синхронизации и показывайте сводное резюме перед запросом.
