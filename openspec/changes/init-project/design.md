## Context

Проект «База знаний» находится на начальной стадии. Требуется создать надежный фундамент, используя современный стек Java/Spring, обеспечивающий масштабируемость и безопасность с первого дня.

## Goals / Non-Goals

**Goals:**
- Инициализировать Spring Boot проект с поддержкой Web MVC и Security.
- Настроить PostgreSQL 18 и систему миграций Liquibase.
- Реализовать базовую структуру пакетов по принципам Clean Architecture.
- Настроить OpenAPI (Swagger) для документирования API.
- Создать базовую ролевую модель (RBAC) и инициализировать администратора.

**Non-Goals:**
- Реализация бизнес-логики редактора документов или поиска.
- Настройка CI/CD пайплайнов (вне рамок данной US).
- Интеграция с внешними системами (Git, Email) — только подготовка конфигурации.

## Decisions

1. **Система сборки: Maven.** Стандарт индустрии для Java-проектов, обеспечивающий предсказуемое управление зависимостями.
2. **Миграции БД: Liquibase.** Выбран из-за гибкости (поддержка XML/YAML/SQL) и возможности легкого отката изменений.
3. **Архитектура: Clean Architecture.** Разделение на слои (domain, application, infrastructure, web) для минимизации связности.
4. **Безопасность: Spring Security (Form Login).** На данном этапе достаточно базовой формы входа для защиты ресурсов.
5. **Документация: springdoc-openapi.** Автоматическая генерация спецификации OpenAPI 3 из кода контроллеров.

## Risks / Trade-offs

- **[Risk]** Несовместимость драйверов с PostgreSQL 18 (так как это очень новая версия). → **Mitigation:** Использовать последнюю стабильную версию JDBC драйвера и протестировать базовые операции.
- **[Risk]** Сложность Clean Architecture для маленького проекта на старте. → **Mitigation:** Создать только необходимые пакеты, не переусложняя структуру до появления реальной логики.
